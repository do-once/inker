# 识别场景状态栏多个操作按钮无响应

> 创建日期：2026-04-23 20:20
> 状态：⚫ Closed
> 优先级：P2-一般
> 影响范围：playground/app — RecognitionScene

## 问题描述

在识别场景中，底部状态栏的"导出 JSON"、"导入 JSON"、缩放"+"和"−"四个按钮点击后没有任何反应。这些按钮在基础场景中工作正常。

## 复现步骤

1. 启动 playground（`pnpm dev`），访问 `http://localhost:3000`
2. 从顶部工具栏的场景选择器切换到"识别调试"
3. 绘制几笔笔画
4. 点击状态栏的"导出 JSON"按钮
5. 期望：触发 JSON 文件下载；实际：无任何响应
6. 同样地，"导入 JSON"、"+"、"−"按钮也无响应

## 技术细节

- **根因**：`RecognitionScene.vue` 的 `onMounted` 中 `registerActions` 只注册了 5 个操作，缺少 `exportJSON`、`importJSON`、`zoomIn`、`zoomOut`
- **涉及文件**：`playground/app/src/scenes/recognition/RecognitionScene.vue:107-111`
- **错误信息**：无报错，`callAction` 在 `App.vue:34-37` 中静默跳过未注册的 action

```typescript
// 当前注册的 actions（缺少 4 个）
registerActions({
  setTool, undo, redo,
  clear: clearAll,
  exportPNG
})

// 对比：BasicScene 注册了全部
registerActions({
  setTool, undo, redo, clear,
  zoomIn, zoomOut,
  exportPNG, exportJSON, importJSON
})
```

## 修复记录

- **修复人**：Claude Code
- **修改文件**：
  - `playground/app/src/scenes/recognition/useRecognitionEditor.ts:213-217` — return 对象增加 `exportJSON`、`importJSON`、`zoomIn`、`zoomOut` 四个透传
  - `playground/app/src/scenes/recognition/RecognitionScene.vue:38-41` — 解构增加 4 个函数
  - `playground/app/src/scenes/recognition/RecognitionScene.vue:107-112` — `registerActions` 增加 4 个操作
- **方案描述**：`useEditor` 已实现全部功能，`useRecognitionEditor` 包装时遗漏了 4 个函数的透传。补上透传并在 `registerActions` 中注册即可。
- **建议验证方式**：运行 `npx playwright test e2e/panel-recognition.spec.ts -g "识别场景"` 验证导出 JSON、导入 JSON、缩放按钮在识别场景中均正常工作
