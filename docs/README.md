# docs — 项目文档

## 目录结构

```
docs/
└── superpowers/
    ├── specs/    # 设计规格文档（brainstorming 产出）
    └── plans/    # 实施计划文档（writing-plans 产出）
```

## 文档命名规范

- 格式：`YYYY-MM-DD-<feature-name>-design.md`（设计）或 `YYYY-MM-DD-<feature-name>.md`（计划）
- 设计文档在 brainstorming 阶段产出，经审查后归档到 `specs/`
- 实施计划在 writing-plans 阶段产出，归档到 `plans/`
